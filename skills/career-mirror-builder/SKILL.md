### L3/L4/L5 Differentiated Aha Moment

**L3 ("My gap became visible")**: Vague anxiety transforms into specific divergence point labels. Emotion: naming relief + mild shock. Mirror Moment complete when user hovers/taps at least 1 divergence point.

**L4 ("I saw how they did it")**: Already knows gaps vaguely. Aha comes from seeing the COHORT'S PATH to closing the gap. Mirror Moment complete when user clicks/hovers the blurred "cohort path detail" area.

**L5 ("I saw what I didn't know I didn't know")**: Unknown-unknown revealed. At least 1 of 3 divergence points must be in a dimension the user did NOT expect. Mirror Moment complete when user dwells 5+ seconds on an unexpected divergence point.

**Design principle**: No need to classify users during onboarding. Show 3 divergence points where at least 1 is predictable (serves L3/L4) and at least 1 is unexpected (serves L5). All levels are served simultaneously.

## PHASE 5: BBAIJJIM PRODUCT AUDIT (Career Mirror Specific)

### 4 Required Architecture Changes for Compliance

1. **Frontier expansion, not gap recycling**: When user self-reports closing a divergence point, system recalculates against the NEXT outcome tier. Never re-display a closed gap. "Congratulations, this gap is closed. At the next level you're targeting, here's what appears."

2. **Graduation design**: When all divergence points for the target outcome are closed, issue a "graduation badge." Graduate's anonymized trajectory is added to the seed cohort. Graduates offered free conversion + "mentor cohort" role. Leaving is fine; contributing is also fine.

3. **Meta-transparency**: "How this analysis works" link always visible at bottom. Explains: "We compare your trajectory with the outcome-achiever cohort using statistical divergence detection. Limitations of this method include..." Full transparency. Builds L5 trust which is a conversion prerequisite.

4. **Capability transfer**: Monthly email "Your trajectory has been updated" includes "How to monitor this gap yourself without the product." Users who can self-track don't need the product for that dimension — but the cohort comparison data remains non-replicable independently, maintaining natural differentiation.

### Metrics to Monitor Post-Launch

| Metric | Healthy | Warning | Action |
|---|---|---|---|
| Monthly gap closure self-report rate | > 30% | < 10% | Product under-utilized |
| Post-closure upper-tier recalculation rate | > 50% | < 20% | Frontier stagnation |
| Post-churn 6-month re-signup rate | > 15% | < 5% | Value extinguished |
| Free share -> paid conversion | > 5% | < 1% | CLG loop broken |

### Healthy PMF vs Addictive Retention: HPMFI Formula

```
HPMFI = Engagement_Score x Capability_Transfer_Score x (1 - Dependency_Score)

Engagement_Score (ES) = min(1.0, M3_Retention / 0.40)
Capability_Transfer_Score (CTS) = (Gap_Reduction/30%)x0.5 + (Output_Increase/1.5x)x0.3 + (Independent_Execution/3.5)x0.2
Dependency_Score (DS) = (Anxiety_Motivation_Ratio x 0.4) + (Excess_Frequency_Flag x 0.3) + (Gap_Stagnation_Flag x 0.3)

HPMFI >= 0.50: Healthy PMF
HPMFI 0.30-0.49: Partial PMF (specific segment issue)
HPMFI 0.15-0.29: At Risk (dependency may be driving retention)
HPMFI < 0.15: No PMF
```

Critical insight: Low retention + high capability transfer = Healthy PMF. High retention + low capability transfer = Addictive retention (NOT PMF).

## PHASE 6: PMF TRACKING

### Core PMF Gate

```
Core_PMF = M3_Retention x Frontier_Motivation_Ratio

M3_Retention = (M3 active users) / (M0 signups)
Frontier_Motivation_Ratio = ("want to see new gaps/frontiers" responses) / (all renewal responses)

Core_PMF >= 0.25 -> PMF achieved
  Example: 40% retention x 65% frontier ratio = 0.26 (pass)
  Example: 50% retention x 40% frontier ratio = 0.20 (FAIL — high retention but unhealthy motivation)

Core_PMF 0.15-0.25 -> Partial PMF (segment analysis needed)
Core_PMF < 0.15 -> No PMF, redesign required
```

### Leading Indicators

**L0 (Interview stage):**
