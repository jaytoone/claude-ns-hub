# YouTube Shorts Script — Korean AI Content Creator

## Overview

Trust-first, 60-second script system for Korean AI engineers building a monetizable audience.
Core principle: **첫 1.5초가 전부다** — hook이 감정을 건드리지 않으면 나머지 58.5초는 없다.
Format: PC 터미널/코드 화면 녹화 (main) + 메바 캐릭터 오버레이 (trust layer) + Fish Audio TTS.

**이 스킬의 핵심 원칙:**
> 스크립트는 **정보 전달** 문서가 아니라 **감정 여정** 설계도다.
> 각 구간은 "무엇을 보여줄까"가 아니라 "시청자가 무엇을 느껴야 하는가"부터 시작한다.

> **TTS 채널 특수 원칙:** 얼굴/목소리 에너지로 스크립트 약점을 보완할 수 없다.
> **스크립트 품질 = 채널 품질 (1:1)**. 문장 구조와 리듬이 곧 감정 전달이다.

## Production Format (Non-Negotiable)

```
메인 트랙: 메바 AI 영상 (9:16)
  도구: Kling AI (subject reference) 또는 MiniMax Hailuo
  입력: 메바 원본 이미지 + 씬별 동작 프롬프트 (스크립트 `영상:` 태그)
  핵심: subject reference로 씬마다 메바 외형 일관성 유지
  씬 구성: 6-8클립 × 5-10초 = 60초

텍스트 오버레이: 증거 레이어 (별도)
  내용: 수치 / 코드 / 벤치마크 결과
  배치: Bold 흰색 중앙 (메바 위 레이어)
  훅 구간 (00-04초): 수치 텍스트 풀스크린 → 메바 없음

감정 큐: (메바: 기본/놀람/흥분/고민) — 스크립트에 명시
  기본: 일반 동작 (걷기, 바라보기)
  놀람: 수치 등장 순간 (눈 커지기, 뒤로 물러나기)
  흥분: 클라이맥스 (점프, 에너지 폭발)
  고민: 문제 제기 (머리 긁기, 갸웃)

오디오: Fish Audio TTS
  문체: TTS 최적화 규칙 필수 적용 (아래 섹션 참조)
  
편집: CapCut
  자막: TTS 자동 생성 → 검토 → Bold 흰색 하단 중앙
  BGM: 없음 (TTS 명료도 우선)
```

## STEP 0 — 스크립트 엔진 (작성 전 필수)

스크립트를 쓰기 전에 아래 4개를 먼저 결정한다. 이게 없으면 내용이 정보 나열이 된다.

### 0-1. 각도 (Angle) — 어떤 방식으로 공격할 것인가

아래 중 하나를 선택한다. 하나의 Shorts = 하나의 각도.

| 각도 | 패턴 | 예시 |
|------|------|------|
| **반직관 충격** | "X라고 생각하는데, 실제로는 Y다" | "파인튜닝이 답이 아닐 수 있다" |
| **상식 파괴** | "모두가 X를 하는데, X는 틀렸다" | "모두가 GPT API를 쓴다. 틀린 선택이다." |
| **수치 폭탄** | "[구체적 수치]... 이게 [충격적 조건]에서 나온 겁니다" | "+14.5%... 이게 코드 40줄에서." |
| **비밀 폭로** | "아무도 안 알려준 [결과]를 직접 만들었다" | "GPT-4가 못하는 걸 직접 만들었다" |
| **공감 분노** | "[모두가 겪는 문제] — 아무도 해결책을 안 알려준다" | "파인튜닝 다음이 뭔지 아무도 안 가르쳐준다" |
| **시대 선언** | "[지금 이 순간]이 전환점이다" | "LLM을 쓰는 시대에서 합치는 시대로" |
| **결과 역전** | "[예상과 반대되는 결과]가 나왔다" | "작은 모델이 GPT-4를 이겼다" |

### 0-2. 갈등 (Conflict) — 무엇이 악당인가

모든 Short에는 갈등 구조가 있어야 한다. 갈등 없이는 서사가 없다.

```
악당 = 기존 방법 / 잘못된 상식 / 시청자가 겪는 고통 / 업계의 관행
영웅 = 나의 발견 / 내가 만든 도구 / 이 영상이 보여주는 것

예시:
- 악당: "파인튜닝에 수천만원 들어간다" → 영웅: "진화 알고리즘으로 모델을 합쳤다"
- 악당: "LLM 성능 올리려면 더 큰 모델이 필요하다" → 영웅: "병합이 파라미터를 안 늘린다"
- 악당: "개인이 모델을 만드는 건 불가능하다" → 영웅: "GPU 1장, 30분"
```

### 0-3. 감정 여정 (Emotional Arc) — 시청자가 구간별로 무엇을 느껴야 하는가

| 구간 | 목표 감정 | 감정 키워드 |
|------|----------|------------|
| Zone 1 (0-4s) | **충격 / 호기심** | "이게 뭐지?" / "말이 돼?" / "잠깐, 다시 봐야 해" |
| Zone 2 (4-12s) | **공감 / 분노** | "나도 이 문제 있었어" / "왜 아무도 안 알려줬지" |
| Zone 3 (12-40s) | **깨달음 / 흥분** | "아 이렇게 되는 거구나" / "이게 진짜야?" |
| Zone 4 (40-53s) | **결정 / 조급함** | "이거 나한테 필요하다" / "빨리 해봐야겠다" |
| Zone 5 (53-60s) | **궁금증 유지** | "다음 편 봐야 한다" / "저장해야지" |

### 0-4. 오픈 루프 스택 (Open Loop Stack) — 각 구간이 닫기 전에 새 질문을 열어야 한다

```
규칙: 각 구간은 이전 질문에 답하면서 동시에 새 질문을 열어야 한다.

Zone 1: "[충격 화면]" → 시청자 질문: "어떻게 이게 가능하지?"
Zone 2: "그 이유 힌트" → 시청자 질문: "그럼 구체적으로 어떻게?"
Zone 3: "메커니즘 설명" → 시청자 질문: "그럼 내가 이걸 할 수 있나?"
Zone 4: "결론 + 갭 강조" → 시청자 질문: "그럼 다음 단계는?"
Zone 5: CTA + 다음 예고

❌ 금지: 각 구간이 완전히 닫히는 것 (= 스킵 유발)
✅ 필수: "이게 다야?"라는 생각이 들게 하고 이미 다음 구간이 시작되어 있어야 함
```

## Script Architecture (60초 고정)

| 구간 | 시간 | 목표 감정 | 기능 | 핵심 규칙 |
|------|------|----------|------|-----------|
| **훅** | 00-04초 | 충격/호기심 | System 1 트리거 | TTS 없음 / 메바 없음 / 화면만 |
| **공감** | 04-12초 | 공감/분노 | 크리에이터 각인 + 감정 연결 | 메바 등장 (놀람/고민) / 인사 없음 / 선언 말투 |
| **증거** | 12-40초 | 깨달음/흥분 | 신뢰 구축 + 메커니즘 | 수치 손글씨 / 다이어그램 실시간 |
| **갭** | 40-53초 | 결정/조급함 | 갭 확인 + 자극 | "이걸 모르면 뒤처진다" 구조 |
| **CTA** | 53-60초 | 궁금증 유지 | 전환 | 타입별 차별화 (아래 참고) |

## 훅 설계 (첫 1.5-4초) — 가장 중요한 구간

**첫 1.5초 = 완성 화면 먼저 + TTS 없음 + 메바 없음 + 감정 폭발**

### 훅 공식 메뉴 (하나 선택)

| 공식 | 패턴 | 예시 A (모델 병합) | 예시 B (다른 소재) |
|------|------|------|------|
| **수치 폭탄** | [숫자] + [충격 조건] | "+14.5%... 코드 40줄." | "RAG 없이 환각 68% 줄었다." |
| **죽은 것들** | [N개] + "DEAD/실패" | "31개 Dead Expert. 절반이 허수." | "프롬프트 100개 테스트. 73개 틀렸다." |
| **역전 결과** | [예상치] vs [실제] | "GPT-4 이겼다. 모델 크기 1/10." | "7B가 70B를 이겼다. 조건이 있다." |
| **시간 충격** | [결과] + "만에" | "LLM 병합. 28분 14초." | "파인튜닝 없이. 설정만 바꿨다. 40분." |
| **금지된 것** | "[X]는 없다고 했다" | "진화 알고리즘은 LLM에 못 쓴다고 했다." | "작은 모델은 한국어 못 한다고 했다." |
| **내가 만든 것** | "[직접 만든 것] 공개" | "직접 만든 진화 LLM 병합 엔진. 공개합니다." | "직접 측정했다. 논문이 틀렸다." |
| **히트맵 충격** | 완성 히트맵 → X표시 | "32개 레이어 중 4개는 놀고 있다." | "Attention 맵 직접 찍어봤다." |
| **before/after** | 완성 before → after | "병합 전. 병합 후. 5.8%p." | "프롬프트 수정 전. 후. 정확도 2배." |

### Frame 1 원칙 (GAP-2)
```
❌ 검정 배경에서 그리기 시작 → Frame 1이 빈 화면 = System 1 미작동
✅ 완성된 결과/숫자를 Frame 1에 즉시 표시 → 지우고 다시 그리기
   "결론 먼저 보여주고, 과정은 본문에서"
```

**금지 패턴:**
- ❌ "안녕하세요" 등 인사
- ❌ "오늘은 ~에 대해 알아보겠습니다"
- ❌ Frame 1이 빈 검정 배경
- ❌ 로고/인트로 영상
- ❌ 감정 없는 정보 제시 ("이것은 X입니다")

## 공감 구간 (04-12초) — 악당을 소환한다

**목표: 시청자가 "나 얘기다"라고 느끼게 한다.**

감정 공감 1문장 + 악당 소환 1문장 = 총 2문장

```
패턴: "[현재 상태 공감] — [있죠 / 그렇죠 / 그랬습니다]"
      "[악당 = 기존 방식의 한계 / 모두가 모르는 것] — [선언]"

O: "파인튜닝은 해봤는데, 그 다음이 뭔지 모르겠다는 느낌 — 있죠."
   "모두가 파인튜닝만 말합니다. 진짜 다음 단계는 아무도 안 알려줍니다."
   
O: "LLM 쓸 줄은 아는데, 직접 만드는 건 엄두도 못 냈죠. 저도 그랬습니다."
   "그런데 진화 알고리즘이 그 벽을 없앴습니다."

X: 감정 없이 바로 수치 설명
X: "저는 AI 엔지니어 [이름]입니다" (자기소개 = 시청자에게 무관심한 신호)
```

**감정 언어 소스:** "신기하다", "이게 된다고?", "나도 할 수 있겠다", "놓치면 뒤처진다", "아무도 안 알려준", "다들 틀렸다", "직접 확인했다"

---

## 증거 구간 (12-40초) — 메커니즘을 공개한다

**목표: "이게 진짜구나"를 느끼게 한다. 단, 정보 나열이 아니라 발견의 과정으로.**

### 서사 구조 (Conflict → Discovery → Implication)

```
[갈등 제시] → [발견] → [의미]

예시:
"기존 파인튜닝은 파라미터를 수정합니다. [드로잉]
진화 알고리즘은 파라미터를 합칩니다. [드로잉]
이 차이가 KoMT-Bench에서 14.5%포인트입니다. [숫자]"

패턴:
- 갈등: "기존 방식은 X를 해야 합니다"
- 발견: "저는 Y를 발견했습니다"  
- 의미: "그 결과가 Z입니다"
```

### 증거 레이어 (신뢰 구축)

1. **수치 (OBSERVED)**: 실제 측정값만 — 추정값 금지
   ```
   O: "KoMT-Bench 7.1/10 (+14.5%)"
   X: "성능이 대략 10% 정도 향상됩니다"
   ```

2. **직접 만든 증거**: 코드/결과물 직접 공개
3. **비교 기준 명확화**: before/after 항상 같은 조건

---

## 갭 구간 (40-53초) — 시청자를 결정으로 밀어붙인다

**목표: "이걸 모르면 뒤처진다"는 느낌 + 행동 의지.**

단순한 인사이트 요약이 아니라 **지자극 + 갭 강조** 구조.

```
패턴: "[결과가 보여주는 것] + [이게 왜 중요한가] + [갭 강조]"

O (강함): "파인튜닝에 수억을 쓰는 팀들이 있습니다.
           이 방법은 그걸 코드 40줄로 합니다.
           모르면 경쟁에서 뒤처집니다."

O (중간): "이걸 알면 LLM 개발 접근 방식이 달라집니다."

X (약함): "이걸 알면 도움이 될 것 같습니다."
X (약함): "정리하자면..."
X (약함): "인사이트를 얻으셨으면 합니다."
```

**금지: 요약으로 끝내기.** 갭 구간은 요약이 아니라 결정을 촉구하는 구간이다.

## TTS 문체 규칙 (Fish Audio 최적화) — 필수

> **왜 중요한가:** TTS는 모든 문장을 동일한 에너지 레벨로 렌더링한다.
> 감정/강조는 문장 구조와 길이로만 만들어야 한다.

```
✅ 분절 (15자 이내)
   "파인튜닝. 다음은 없다."
   "GPU 한 장. 30분."
   
✅ 수치 단독 문장
   "십사 점 오 퍼센트."
   "코드 사십 줄."
   
✅ 명사형 종결
   "코드 사십 줄. 실행. 끝."
   "진화 알고리즘. 적용. 완료."

❌ 연결어미
   "파인튜닝을 했는데 그 다음이 없다고들 하죠" → 금지
   
❌ 감탄사/추임새
   "어, 근데..." / "맞죠?" / "사실은요..." → 금지
   
❌ 긴 단문 (15자 초과)
   "이 방법은 기존 파인튜닝 방식과는 완전히 다른 접근입니다" → 쪼개기
```

**리듬 패턴 (강-약-강):**
```
"진화 알고리즘.     ← 강 (핵심 단어)
 모델을 합칩니다.   ← 약 (동사)
 GPT-4를 이겼습니다." ← 강 (결과)
```

## 메바 감정 큐

스크립트에 `(메바: [상태])` 태그로 명시 → CapCut 편집 시 메바 이미지 교체 타이밍

| 큐 | 사용 장면 | 예시 |
|----|---------|------|
| `(메바: 기본)` | 일반 설명 구간 | 코드 설명, 개념 소개 |
| `(메바: 놀람)` | 수치/결과 등장 | "+14.5%" 화면 노출 시 |
| `(메바: 흥분)` | 클라이맥스, 핵심 포인트 | "이게 전부입니다" |
| `(메바: 고민)` | 문제 제기, 질문 장면 | "파인튜닝 다음은?" |

**메바 등장 규칙:**
- 훅 구간 (00-04초): 메바 없음 — 화면 임팩트만
- 00:04 이후: 메바 등장, 이후 감정 큐에 따라 교체
- 위치: 우하단 20%, 원형 크롭

## 나레이션 스타일

```
선언 말투 (O):  "파인튜닝까지 해봤으면, 이미 여기까지 와 있는 거다."
설명 말투 (X):  "오늘은 파인튜닝에 대해 설명해 드리겠습니다."

단호하게 (O):   "GPU 한 장. 30분."
부드럽게 (X):   "GPU를 한 장만 써서 약 30분 정도면..."

결론 먼저 (O):  "이 차이가 5.8%포인트입니다." → 이유 설명
이유 먼저 (X):  설명 → 설명 → 결론

자극적으로 (O): "아무도 안 알려준 방법입니다."
중립적으로 (X): "한 가지 방법을 소개하겠습니다."
```

**말하는 속도**: 천천히, 한 문장마다 0.5초 침묵 허용
**자막 규칙**: 핵심 단어/수치만 → Bold 강조 (노란색 or 흰색)

## Script Types

### Type A — 신뢰 구축 (시리즈 1-10편)
- **목적**: "직접 만드는 사람" 각인
- **CTA**: 다음 편 예고 (구독 유도) / 강의 언급 없음
- **후크 패턴**: 수치 폭탄 / 역전 결과 / 죽은 것들
- **Loop Closure 규칙 (GAP-3)**: mini-close + bridge
  → "이걸 알면 [결과]가 달라진다" 로 이번 편 내부 결론 제공
  → "다음 편에서 [구체적 예고]" 로 bridge (SPIKE Embed 유지)

### Type B — 가능성 (시리즈 5편 이후)
- **목적**: "나도 할 수 있겠다" 감각 이식
- **CTA**: 강의 소프트 언급 ("코드 먼저 보고 결정해도 됩니다")
- **후크 패턴**: 시간 충격 / 내가 만든 것

### Type C — 시대 선언 (시리즈 10편 이후)
- **목적**: 세계관 공유 → 커뮤니티 형성
- **CTA**: 강의 직접 언급 + 얼리버드
- **후크 패턴**: 시대 선언 / 공감 분노

### Type D — 제품 연계 (신뢰 충분 후)
- **목적**: SaaS/툴 자연스러운 소개
- **CTA**: 제품 링크 (VIDraft 등)

## CTA 설계 규칙

### 갭 닫기 순서 (절대 역순 금지)
```
1단계 (1-5편):  Trust → "다음 편에서 보여드립니다"
2단계 (5-10편): Attainability → "코드 먼저 보고 결정해도 됩니다"
3단계 (10편+):  Curriculum → 강의 직접 언급
4단계 (검증 후): Urgency → 얼리버드 마감
```

**절대 하지 말 것:**
- ❌ 1편부터 강의 CTA
- ❌ 가짜 카운트다운
- ❌ "코드 전부 공개"
- ✅ "핵심 개념 코드 공개, 전체 실습은 강의에서"

## 화면 구성 가이드 (메바 + 화면 녹화)

```
배경: 터미널 검정 / VS Code 다크테마 / 수치 텍스트
텍스트 강조:
  노란색: 핵심 수치, 결론
  빨간색: 문제점, 기존 방식
  초록색: 해결책, 결과

메바 오버레이:
  위치: 우하단 20%
  형태: 원형 크롭
  등장: 00:04~
  감정 큐 전환: 스크립트 `(메바: __)` 태그 기준

레이아웃 원칙:
  - 화면 중앙 70%: 코드/결과/수치
  - 우하단 20%: 메바
  - 자막: 하단 중앙 Bold 흰색

**화면 녹화 준비:**
- 폰트 크기 키우기 (터미널 16pt+, 전체 화면 가독성)
- 불필요한 탭/파일 닫기 (집중도)
- Frame 1 = 결론 수치 즉시 노출 (터미널에 미리 출력해두기)

## 스크립트 작성 체크리스트

**STEP 0 완료 여부 (스크립트 쓰기 전):**
- [ ] 각도 (Angle) 결정: 반직관 충격 / 상식 파괴 / 수치 폭탄 / 비밀 폭로 / 공감 분노 / 시대 선언 / 결과 역전
- [ ] 갈등 구조 명확: 악당 = [기존 방식], 영웅 = [내 발견]
- [ ] 감정 여정 설계: Zone1(충격) → Zone2(공감) → Zone3(깨달음) → Zone4(결정) → Zone5(궁금증)
- [ ] 오픈 루프: 각 구간이 새 질문을 열고 있음

**훅 (00-04초):**
- [ ] TTS 없음 / 메바 없음 / 화면만
- [ ] Frame 1에 완성 결과/숫자 즉시 노출
- [ ] 훅 공식 선택 완료 (8개 중 하나)
- [ ] "이게 뭐지?"를 유발하는가? (YES/NO)

**공감 (04-12초):**
- [ ] 감정 공감 1문장 포함
- [ ] 악당 소환 1문장 포함
- [ ] 인사/자기소개 없음
- [ ] `(메바: 고민)` or `(메바: 놀람)` 큐 삽입

**증거 (12-40초):**
- [ ] 갈등 → 발견 → 의미 구조
- [ ] OBSERVED 수치 최소 1개
- [ ] "정보 나열"이 아닌 "발견 과정" 형태인가?

**갭 (40-53초):**
- [ ] 요약이 아닌 결정 촉구 구조
- [ ] "이걸 모르면 뒤처진다" 또는 동급 긴장감

**CTA (53-60초):**
- [ ] 편수에 맞는 CTA 단계
- [ ] 다음 질문을 열어두는 예고

**TTS 문체:**
- [ ] 전체 문장 15자 이내 분절 확인
- [ ] 수치는 단독 문장으로 분리
- [ ] 연결어미 없음 (했는데, 그래서, 그러면 → 제거)
- [ ] 감탄사/추임새 없음

**편집:**
- [ ] 메바 00:04부터 등장
- [ ] 메바 감정 큐 타이밍 명시
- [ ] 자막 핵심만 Bold 강조

## 시리즈 설계

```
편수    내용                          CTA
1-5편   핵심 개념 (Type A)            다음 편 예고
5-10편  심화 개념 + 가능성 (A+B)      소프트 강의 언급
10-15편 결과 증명 + 비교 (A+C)        강의 직접 CTA
15-20편 커뮤니티 형성 (C+D)           얼리버드 + 제품 연계
20편+   알고리즘 트랙션 기준 (최소 20편)
```

5편 공개 후: 강의 요청 댓글/DM 카운팅

      10건 미만 → 포맷/소재 개선 후 리메이크
```
