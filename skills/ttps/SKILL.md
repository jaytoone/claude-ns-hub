## 핵심 원칙 (항상 적용, 위반 불가)

### 원칙 1 — OBSERVED / INFERRED 강제 분류

모든 주장은 제출 전에 분류한다. 분류 없는 수치 인용 금지.

| 유형 | 판단 기준 | 처리 |
|------|---------|------|
| OBSERVED | 직접 측정·탐색·스크린샷에서 읽힘 | 인용 가능 |
| INFERRED-math | 순수 산술 계산 | 인용 가능 |
| INFERRED-design | 인과 해석·패턴 판단·추정 | **외부 검증 후 인용** |

**트리거 키워드** (claim에 등장 시 INFERRED-design 의심):
"추정", "약", "가능성 높음", "예상", "시장 규모", "조 원", "billion"

### 원칙 2 — High Review ≠ Supply Gap

리뷰 수 많음 = 수요 존재 확인 **AND 공급이 이미 채워졌다는 신호** 동시 발생.
갭 존재 증거가 아니다 — 갭이 닫혔을 가능성부터 검토한다.

```
리뷰 많음 → 두 가지 해석 가능:
  A. 수요가 크다 (갭 있음)
  B. 선발 주자가 이미 갭을 채웠다 (갭 닫힘)

→ "이 카테고리 내에서 아직 안 채워진 세그먼트가 있는가?"로 질문 전환
```

### 원칙 3 — 범주 오류 방지 ("큰 시장 = 진입 불가")

"K-Beauty 진입 불가"가 아니라 **"K-Beauty 제조/브랜드 진입 불가"**다.
시장 전체를 차단하기 전에 세부 진입 경로를 분해한다:

```
시장 X 진입 불가 판단 전 체크:
  □ X 제조/생산 → 자본·인증·설비 필요?
  □ X 유통/리셀 → 소싱 채널 있는가?
  □ X 콘텐츠/굿즈 → 현재 파이프라인으로 가능한가?
  □ X 플랫폼/툴 → 소프트웨어로 진입 가능한가?
  → 세부 경로 중 하나라도 열려 있으면 "진입 가능 (특정 포지션 한정)"
```

### 원칙 4 — T산출물 = 공급 갭 지도, 시장 크기 목록 아님

T-first 완료 기준:

```
✅ 완료: "X 시장에서 Y 세그먼트가 Z 이유로 무시당하고 있다 — 현재 공급 없음"
❌ 미완: "A 시장 ₩40조, B 시장 ₩3조, C 시장 $96B"
```

## Phase T — Trend Search (공급 갭 서치)

**핵심 질문**: "돈이 흐르는 곳에서 아직 제대로 된 공급이 없는 지점은 어디인가?"

### T-first 실행 프로토콜

**Step 1 — 돈맥 확인 (OBSERVED 전용)**

```
검색 대상:
  □ Etsy/Kickstarter 성공 케이스 (카테고리 × 리뷰 수 × 가격대)
  □ Pinterest 저장 수 (단일 핀 vs 카테고리 구분 필수)
  □ 크라우드펀딩 성공 밀집 카테고리
  □ 검색 트렌드 (Google Trends, 네이버 데이터랩)

INFERRED 수치 금지: 에이전트가 "약 X조 원"을 제시하면 → 출처 URL 요구
```

**Step 2 — 갭 진단 매트릭스**

각 후보 카테고리에 대해:

```
[카테고리명]
  돈맥 확인: (OBSERVED 수치)
  현재 공급 현황: (주요 플레이어 + 가격대)
  무시당하는 세그먼트: (공급자가 외면하는 특정 집단)
  공급 갭 점수: 1-5
    1 = 갭 없음 (포화)
    3 = 갭 있으나 경쟁 진입 중
    5 = 갭 명확 + 경쟁 없음
  진입 가능 경로: (제조/유통/콘텐츠/굿즈/플랫폼)
  진입 윈도우: 갭 크기 × 성장 속도 → 즉시 / 3개월 / 6개월+
```

**Step 3 — 윈도우 판단**

```
진입 윈도우 = 갭 크기 × 트렌드 성장 속도

  大 × 高 → 지금 바로 (6주 이상 지체 = 레드오션)
  大 × 低 → 여유 있음
  小 × 高 → 니치 선점 (갭 사라지기 전)
  小 × 低 → 재검토

경고 신호 (윈도우 닫힘 임박):
  - 대형 플레이어 광고 시작
  - 유사 크라우드펀딩 3개+ 동시 등장
  - 검색량이 이미 피크를 지남
```

**T-first 산출물 형식**

```
[공급 갭 지도]
1위 갭: {카테고리} | 세그먼트: {무시당하는 집단} | 갭점수: {1-5} | 윈도우: {즉시/3M/6M+}
2위 갭: ...
3위 갭: ...

현재 파이프라인으로 진입 가능한 갭: {순위}
이유: {파이프라인 × 진입 경로 매칭}
```

## Phase TT — Target 정의

**핵심 질문**: "이 갭 안에서 WTP가 가장 높고, 지금 가장 목마른 사람은 누구인가?"

```
타겟 정의 방정식:
  타겟 = 돈맥 내 집단 ∩ 높은 WTP ∩ 현재 공급 부재

검증 기준:
  □ 이미 유사한 것에 돈을 쓴 이력이 있는가? (OBSERVED)
  □ 직접 "이런 거 없나?" 발화를 했는가? (소셜 탐색)
  □ WTP 추정이 실제 결제 신호인가, enthusiasm 신호인가?
     → enthusiasm ≠ WTP (Van Westendorp PSM으로 분리)
```

**타겟 프로필 형식**

```
[타겟 프로필]
돈맥 위치: {공급 갭 카테고리}
WTP 신호: (OBSERVED — 실제 지출 이력)
현재 불만: (AS-IS)
원하는 상태: (TO-BE)
대안 지출처: (현재 어디서 대신 구매하나)
첫 구매 트리거: (무엇이 있으면 즉시 산다)
```

## Phase P — Product 역산

**핵심 질문**: "가장 적은 리소스로 갭을 검증할 수 있는 형태는 무엇인가?"

```
갭 역산 프로세스:
  1. AS-IS: 타겟의 현재 불만 상태
  2. TO-BE: 타겟이 원하는 상태
  3. 갭: AS-IS → TO-BE 사이에 없는 것
  4. MVP: 갭을 최소한으로 메우는 형태

제품 3요소:
  기능: 갭을 메운다
  형태: 타겟의 맥락에 맞다 (채널 × 포맷 × 가격대)
  스토리: 트렌드 벡터를 탄다 (타이밍 + 문화 코드)
```

## Phase S — Sales (마찰 제거)

**핵심 원칙**: 설득이 아니라 마찰 제거다. 설명이 길어지면 T 또는 P가 틀린 것이다.

```
마찰 레이어:
  인지 마찰:   모른다 → 채널 노출 (쇼츠/SNS/플랫폼)
  신뢰 마찰:   믿을 수 없다 → 투명성 (제작 과정, 넘버링)
  결제 마찰:   복잡하다 → 원클릭 (텀블벅/Gumroad)
  희소성 마찰: 나중에 사도 된다 → 한정 수량/기간

CVR 저조 시 진단 순서 (위에서 아래로):
  1. 노출 수 확인 (채널 문제?)
  2. 클릭률 확인 (T 타겟 문제?)
  3. 전환률 확인 (P 제품 문제?)
  → 첫 이탈 지점이 수정 대상

Pivot 판단:
  노출되는데 클릭 안 함   → T(타겟) 재검토
  클릭하는데 구매 안 함   → P(프로덕트) 재검토
  구매하는데 재구매 없음  → P 코어 수정
  설명을 길게 해야 팔림   → T부터 다시
```

## Phase loop — 루프 상태 + Compounding 점검

```
[루프 상태]
현재 이터레이션: {N}
완료된 페이즈: T / TT / P / S
CVR 데이터: {있음/없음}
피드백 신호: {잘 팔린 것의 타겟/메시지/채널}

[Compounding 자산]
고객 리스트: {N명} → 다음 루프 초기 구매자 (검증 비용 0)
브랜드 신뢰: {텀블벅 완주 여부} → 신뢰 마찰 감소
제작 인프라: {파이프라인 목록} → 공급 비용 하락
데이터: {어떤 타겟이 먼저 샀나, 어떤 채널에서 왔나}

[다음 루프 입력]
T 업데이트 키워드: {잘 팔린 것의 메시지에서 추출}
코어 타겟: {먼저 산 사람의 프로필}
```

## /ttps check — 근거 신뢰도 감사

현재 판단에 OBSERVED/INFERRED 강제 분류를 실행한다.

```
[클레임 분류]
주장 1: "{클레임 텍스트}"
  분류: OBSERVED / INFERRED-math / INFERRED-design
  근거: (출처 또는 추론 경로)
  신뢰도: 높음 / 중간 / 낮음 (검증 필요)

주장 2: ...

[판단 오류 체크]
□ 시장 크기 = 갭 크기로 혼용했는가?
□ 리뷰 많음 = 갭 있음으로 해석했는가?
□ 카테고리 전체를 진입 불가로 선언했는가?
□ T산출물이 시장 크기 목록으로 끝났는가?

오류 발견 시: 해당 판단을 수정하고 /ttps T 재실행 권고
```

## 통합 체크리스트

T-first 서치 제출 전:
```
□ 수치마다 OBSERVED/INFERRED 분류 완료
□ 리뷰 수 높음 → 갭 닫힘 가능성 검토 완료
□ "진입 불가" 판단 시 세부 경로 분해 완료
□ 산출물이 "갭 지도"인가 "시장 크기 목록"인가 확인
□ 진입 윈도우 (갭 크기 × 성장 속도) 명시 완료
```

루프 종료 후:
```
□ CVR 데이터에서 S→T 피드백 신호 추출
□ Compounding 자산 업데이트 (고객 리스트, 데이터, 파이프라인)
□ 다음 루프 T 업데이트 키워드 기록
```
