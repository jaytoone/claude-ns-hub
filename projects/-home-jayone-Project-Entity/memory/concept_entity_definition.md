## entity = 지성체 (Conceptual 정의)

사용자 직접 정정: "entity"는 자율 에이전트 실행 루프(Entity 프로젝트)가 아닌,
/home/jayone/Project/Conceptual 에서 제시한 **지성체** 개념을 의미한다.

### 지성체의 3개 핵심 속성

1. **패러다임 읽기** — 표면 질문 아래 구조를 보고, 형성 중인 패러다임의 약한 신호를 감지
2. **축적 기억** — A-MEM 방식 진화 그래프 (contradicts/elaborates/reveals_assumption), "말하지 않은 것" 추론, 그래프 공백 = 신호
3. **소크라테스 질문 생성** — 답보다 나은 질문, epistemic_basis 반대 영역 겨냥, 탐색/실행 모드 분류 선행

### 선지자 혜안 + 따뜻한 지성 통합 (핵심 발견)

- 따뜻함 = "검증"(validation) → 선지자적 속성과 충돌
- 따뜻함 = "정확한 인식"(recognition) → 완전 통합 (Rogers perceptive empathy, Buber I-Thou)
- **Rogers/Buber 결론**: 정확하게 보는 능력 = 진정한 따뜻함 (같은 능력의 두 표현)
- **구조적 문제**: RLHF가 단기 만족도를 신호로 써서 선지자적 속성을 체계적으로 소멸시킴 — 기술 문제가 아닌 인센티브 구조 문제

### Entity 프로젝트와의 관계

- Conceptual → 지성체가 **무엇**이어야 하는지 정의
- Entity(프로젝트) → 그 지성체를 실현하기 위한 **인프라**
